# 何峰-专用记事本

一个功能完整的个人任务管理和图片管理应用，支持实时数据同步。

## 功能特点

### 📝 任务管理
- 四象限任务管理（紧急重要、重要不紧急、紧急不重要、不紧急不重要）
- 任务优先级设置
- 任务完成状态跟踪
- 任务分类和标签
- 批量操作（删除选中、删除已完成、恢复已删除）

### 🖼️ 图片管理
- 图片上传和压缩
- 图片分类管理
- 图片搜索和筛选
- 图片预览和编辑

### 📊 数据分析
- 任务分布图表
- 完成情况统计
- 优先级分析
- 效率趋势跟踪

### 🔐 安全特性
- 密码保护登录
- 会话超时自动退出
- 数据验证和清理
- 输入防XSS攻击

### 💾 数据存储
- Supabase云端数据库
- 本地存储回退机制
- 实时数据同步
- 离线工作支持

## 技术架构

### 前端技术
- HTML5 + CSS3 + JavaScript (ES6+)
- Tailwind CSS 样式框架
- Chart.js 数据可视化
- SortableJS 拖拽排序

### 后端服务
- Supabase (PostgreSQL + 实时订阅)
- 行级安全策略 (RLS)
- 存储桶管理

### 数据模型
- 任务表 (tasks)
- 图片表 (images) 
- 操作日志表 (operation_logs)

## 安装部署

### 本地运行
1. 下载所有文件到同一目录
2. 确保文件结构如下：
notebook-app/
├── index.html
├── css/
│ └── style.css
├── js/
│ ├── config.js
│ ├── utils.js
│ ├── database.js
│ ├── auth.js
│ ├── tasks.js
│ ├── images.js
│ ├── analytics.js
│ └── app.js
└── README.md


3. 用浏览器打开 `index.html`

### GitHub Pages 部署
1. 创建GitHub仓库
2. 上传所有文件到仓库
3. 在仓库设置中启用 GitHub Pages
4. 访问生成的页面地址

### Supabase 配置
应用已预配置Supabase连接，如需自定义：
1. 修改 `js/config.js` 中的配置
2. 在Supabase中创建对应的数据表

## 使用说明

### 首次使用
1. 默认密码：`6858`
2. 登录后建议立即修改密码
3. 开始添加任务和管理图片

### 任务管理
1. **添加任务**：在侧边栏或各象限输入任务内容
2. **设置优先级**：选择任务的重要紧急程度
3. **完成任务**：点击任务前的复选框
4. **编辑任务**：点击任务旁的编辑按钮
5. **删除任务**：点击任务旁的删除按钮

### 图片管理
1. **上传图片**：拖拽或点击上传区域
2. **查看图片**：点击眼睛图标预览
3. **编辑信息**：点击编辑按钮修改名称和描述
4. **删除图片**：点击删除按钮

### 数据分析
1. 切换到"数据分析"视图查看统计
2. 查看任务分布和完成情况
3. 分析工作效率趋势

## 安全注意事项

1. **密码安全**：定期更换密码，不要使用简单密码
2. **会话管理**：不活动15分钟后自动退出
3. **数据备份**：重要数据建议定期导出备份
4. **浏览器存储**：清除浏览器数据会丢失本地缓存

## 故障排除

### 常见问题
1. **无法登录**：检查密码是否正确，尝试默认密码 `6858`
2. **数据不同步**：检查网络连接，应用会自动切换到本地模式
3. **图片上传失败**：检查图片格式和大小（支持JPEG、PNG、GIF、WebP，最大5MB）
4. **图表不显示**：刷新页面或检查网络连接

### 技术支持
如遇技术问题，请检查：
1. 浏览器控制台错误信息
2. 网络连接状态
3. Supabase服务状态
